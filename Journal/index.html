This article provides some tips for effective game design. The article mentions a concept called dark-UX,where desginers will intentionally put promotional interactive elements in areas on the screen that is easiest to reach so that users are likely to notice them and click on them by either curiosity or by mistake. This is common in a lot of game design because it is proven to maximize revenue. These elements includes: store buttons, more apps, or ads.
Then for pop-ups, the article suggested to avoid the "X" button because this might suggest this pop-up is something negative that the users will want to close. Instead, replace it with actionable buttons such as "done", or "next step" will give more positive suggections such as the completion of a task. The important thing is to ask the users to take a decision. In terms of user choice in pop-ups, the decision that is good for the game would be placed on the right side, becasue the majority of users are right handed; the decision that is bad for the game is usually on the left side. The article also has tips for other promotional parts of the game that might represent the company in a way, for example in "rate us" dialogues or store dialogues, it is good to have a presentor that is looking at the users; and the action we want the user to do should be a postive color. These parts of the game if done well can be free promotion for the company.

Best Practices for Modals

Feb 15 2024

Modals are pop-up windows that grab the user's attention, and it temporarily takes up the whole screen space until the user closes the modal. Modals can have advantages in several ways, such as it saves screen real estate, because it doesn't take up spaces in the actual webpage; and it is a good way to direct the user's attention to some information. Modal windows could be annoying as well, as it interrupts the flow of what the user is trying to do.
Here are some best practices for designing modals: All of the buttons should be actionable and understandable, there should be a clear action button that direct the user to a next step; there should be an escape hatch, so that users have the option to leave; its beeter practice if the modal is initiated by some action the user makes, such as clicking on a button, this way the modal is more integrated in user flow. Finally modals doesn't match well with mobile, in mobile interface its better to find alternatives such as scroll bar or drop-down menu. This journal is in response to this article.

Best Practices for Form Design

Feb 15 2024

Forms are a vital tool for data collection. UI/UX designers often use it for surveys; business of any fields use forms to gather data; the job application process is also leaning increasingly towards a remote setting. It is important for designers to learn to avoid usability errors and craete forms that are easier to folllow.
The article discusses 20 best practices for form design. Each of these practices are considering the user's workflow, they are compartmentalizing information into steps, priority levels, and necessities. One practice I didn't notice before is the placement of inline texts; it is better to place the labels at the top of the text field, we also need to pay attention to alignment, and margin between the labels and text fields. The text fields it self should give an idication of the amount of text expected from the users. Dropdown menues are not great for forms and it is something we should avoid. The form should have visible priorities, we do this by highlighting certain text bubbles, and by having an action button.
